# Documentation index

This directory contains the product, engineering, security, and operational documentation for Payflow.

## Product and delivery

- [Project scope](project-scope.md): product goals, users, implemented capabilities, boundaries, and success measures.
- [Delivery plan](project-plan.md): phased roadmap, priorities, acceptance criteria, and known technical debt.

## Engineering

- [Architecture](architecture.md): system context, components, data flow, authorization model, assistant design, and deployment.
- [Security](security.md): security controls, threat considerations, and secure engineering requirements.
- [Data security and retention](data-security.md): sensitive-data classification, retention, and storage requirements.
- [Privacy](privacy.md): privacy workflows and data-subject handling.
- [Operations](operations.md): production readiness, monitoring, incident response, backup, and recovery.

## Documentation principles

- Clearly label implemented behavior versus planned work.
- Treat source code, migrations, and automated tests as the final authority when documentation and behavior differ.
- Update scope, architecture, and operational runbooks whenever a change affects product boundaries, trust boundaries, or support procedures.
